信息收集是渗透测试的第一步，也是最重要的环节。全面的信息收集能够发现更多的攻击面和潜在漏洞。

## 目标识别

### 域名与子域名
- 主域名识别和关联域名发现
- 子域名枚举（主动扫描、被动收集、证书透明度、DNS记录）
- 历史域名和过期域名
- 域名劫持和子域名接管

### IP 资产
- IP 段识别和 C 段扫描
- CDN 识别和源站探测
- 云服务识别（AWS、Azure、阿里云、腾讯云）
- IP 归属和地理位置

### 端口与服务
- 端口扫描策略（全端口、常用端口、隐蔽扫描）
- 服务识别和版本探测
- 服务指纹识别
- 非标准端口服务发现

## Web 资产收集

### 网站信息
- Web 服务器识别（Nginx、Apache、IIS、Tomcat）
- CMS 识别和版本探测
- 框架识别（Spring、Django、Laravel、ThinkPHP）
- 中间件识别和配置信息

### 目录与文件
- 目录爆破和敏感路径发现
- 备份文件和源码泄露
- 配置文件和日志文件
- API 接口发现

### 前端信息
- JavaScript 文件分析
- 注释信息和调试信息
- API 端点和参数
- 前端框架和库版本

## OSINT 开源情报

### 搜索引擎
- Google Hacking（高级搜索语法）
- Shodan、Censys、Fofa、ZoomEye 等网络空间搜索引擎
- GitHub 代码搜索（泄露的密钥、配置、凭证）
- Pastebin 和代码分享站点

### 社交媒体
- 员工信息收集（LinkedIn、微博、Twitter）
- 组织架构和人员关系
- 技术栈和项目信息
- 社交工程情报

### 企业信息
- 工商信息和企业关系
- 招聘信息（技术栈、系统架构）
- 供应商和合作伙伴
- 新闻和公告

### 暗网情报
- 暗网论坛和市场
- 数据泄露和凭证库
- 漏洞交易和工具
- 威胁情报

## 技术信息收集

### DNS 信息
- DNS 记录枚举（A、AAAA、MX、TXT、NS、CNAME）
- DNS 区域传送
- DNS 历史记录
- DNSSEC 配置

### SSL/TLS 证书
- 证书信息和颁发机构
- 证书透明度日志
- 证书关联的域名
- 证书配置和漏洞

### 邮件系统
- 邮件服务器识别
- SPF、DKIM、DMARC 记录
- 邮件地址收集
- 邮件系统漏洞

### 网络架构
- 路由追踪和网络拓扑
- 防火墙和安全设备识别
- 负载均衡和反向代理
- 网络边界和信任域

## 漏洞信息

### 已知漏洞
- CVE 漏洞库搜索
- 历史漏洞和补丁情况
- 漏洞利用代码（Exploit-DB、GitHub）
- 0day 和 1day 情报

### 配置问题
- 默认配置和弱配置
- 不安全的服务和协议
- 敏感信息泄露
- 权限配置错误

## 工具与技术

### 自动化工具
根据目标特征选择合适的工具：
- 子域名枚举工具
- 端口扫描工具
- 目录爆破工具
- 信息收集框架

### 被动收集
- 避免主动扫描触发告警
- 利用第三方平台和缓存
- 历史数据和归档信息
- 公开数据源

### 主动探测
- 控制扫描速度和频率
- 使用代理和分布式扫描
- 伪装扫描来源
- 规避检测和封禁

## 信息整理

### 资产清单
- 建立完整的资产列表
- 标注资产的重要性和价值
- 识别关键资产和薄弱点
- 绘制攻击面地图

### 情报分析
- 关联分析和数据挖掘
- 识别攻击路径
- 评估风险和优先级
- 制定测试计划

---

根据目标的特点和测试范围，灵活选择信息收集的深度和广度，确保在合法授权范围内进行。
