子域名枚举是信息收集中的重要环节，能够发现更多的攻击面和潜在目标。

## 枚举方法

### 被动收集
- 证书透明度日志（crt.sh、Censys）
- 搜索引擎（Google、Bing）
- DNS 聚合平台（VirusTotal、SecurityTrails）
- 第三方 API（Shodan、Censys、Fofa）
- 历史 DNS 记录

### 主动扫描
- DNS 字典爆破
- DNS 区域传送（AXFR）
- DNS 递归查询
- 排列组合生成
- 智能字典生成

### 证书分析
- SSL/TLS 证书中的 SAN 字段
- 证书透明度日志挖掘
- 通配符证书识别
- 证书关联分析

## 工具选择

根据目标规模和时间要求选择合适的工具和策略，结合多种方法提高覆盖率。

## 结果验证

### 存活性检测
- DNS 解析验证
- HTTP/HTTPS 探测
- 端口扫描确认
- 服务识别

### 去重与过滤
- 相同 IP 的子域名
- CDN 和云服务
- 无效和错误的记录
- 泛解析识别

## 深度挖掘

### 递归枚举
- 发现的子域名作为新的目标
- 多级子域名枚举
- 关联域名发现

### 特殊记录
- 内部域名和测试域名
- 开发和预发布环境
- 管理后台和 API 接口
- 历史和废弃域名

---

子域名枚举需要结合多种方法，被动收集为主，主动扫描为辅，确保覆盖率的同时避免触发防护。
